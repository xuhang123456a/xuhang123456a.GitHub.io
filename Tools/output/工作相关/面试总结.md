# 面试总结

非常好的面试题概括,我来详细解答一下:面向对象编程(OOP):OOP 是一种编程范式,它将程序视为一系列相互作用的对象。它的基本特征包括:

封装:将数据和相关的方法封装在一个类中,隐藏内部实现细节。继承:子类可以继承父类的属性和方法,实现代码的重用。多态:不同类的对象可以使用相同的方法名称,但执行不同的操作。类、对象、继承、多态:类是对象的蓝图或模板,定义了对象的属性和行为。对象是类的实例,是程序运行时的具体体现。继承允许子类继承父类的属性和方法,实现代码的重用。多态使得不同类的对象可以使用相同的方法名称,但执行不同的操作。抽象类和接口:抽象类定义了一组通用的属性和方法,但不能实例化。它可以包含具体方法。接口定义了一组方法签名,但不包含方法实现,用于定义规范。接口不能包含具体方法。C# 语言特性:var 关键字用于声明隐式类型的局部变量,编译器会根据初始值推断变量类型。using 语句用于管理资源的生命周期,如文件流、数据库连接等。资源会在 using 块结束时自动释放。ref 和 out 关键字:

ref用于传递引用类型参数,要求参数必须先被赋值。out用于传递输出参数,不要求参数先被赋值。装箱和拆箱:装箱是将值类型转换为引用类型的过程,会造成性能开销。拆箱是将引用类型转换回值类型的过程,也会造成性能开销。垃圾回收:C#采用标记-清除算法进行垃圾回收。Unity在C#基础上提供了自己的垃圾回收实现,使用增量式收集和分代收集策略,可以降低单帧内GC造成的性能开销。泛型约束和协变/逆变:泛型约束用于限制泛型类型参数的类型。协变和逆变是关于泛型类型参数的替换规则。异步编程:异步编程允许程序在执行耗时操作时,不阻塞主线程,而是继续执行其他任务。async/await关键字用于定义和等待异步操作。**Task/Task<T>**类用于表示和管理异步操作。进程、线程、协程:进程是操作系统分配资源的基本单位。线程是进程中的执行单元,多线程可以提高程序的并发性能。协程是一种用户态的轻量级线程,可以在单线程中实现准并发。集合和LINQ:C#提供了丰富的集合类型,如List<T>、Dictionary<K,V>、HashSet<T>等。LINQ(Language Integrated Query)是一种用于查询和处理数据的统一模型,可用于统一查询不同类型的数据源。示例: var youngPeople = persons.Where(p => p.Age < 30).OrderBy(p => p.Name);委托和事件:委托是一种引用方法的类型,用于将方法作为参数传递或赋值给变量。事件是一种消息传递机制,用于实现观察者模式。事件由委托类型定义。异常处理:try-catch-finally块用于处理可能引发异常的代码。try块包含可能抛出异常的语句,catch块处理特定类型的异常,finally块无论是否有异常发生都会执行。
